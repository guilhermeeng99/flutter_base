# FlutterBase — Copilot Instructions

Gamified Flutter learning platform (Duolingo-style) with interactive quizzes, XP, streaks, levels, daily challenges, and a progress dashboard.

---

## Stack

| Aspect | Detail |
|---|---|
| **Flutter SDK** | ^3.11.0 |
| **State Management** | Bloc/Cubit (`flutter_bloc`) |
| **DI** | Factory constructors in Cubits (manual wiring) |
| **Routing** | `go_router` (declarative, path-based) |
| **Storage** | Local-only (in-memory). No remote API, no backend. |
| **Code Generation** | `freezed` + `json_serializable` (models), `slang` (i18n) |
| **Linting** | `very_good_analysis` (strict) |
| **Fonts** | `google_fonts` — JetBrains Mono (headings/buttons), Inter (body), Fira Code (code) |
| **i18n** | `slang` / `slang_flutter` — JSON-based, type-safe, locales: `en` and `pt` |
| **Theme** | Dark-only Material 3, `AppColorsExtension` (ThemeExtension) + `AppTextStyles` + `AppTheme` |
| **Deploy** | GitHub Pages via `deploy_web.bat` |
| **Monorepo** | `melos` for scripts (`slang`, `generate_assets_folders`, `run_release_deploy`) |

## Architecture — Clean Architecture (Feature-First)

```
lib/
├── app/
│   ├── assets/
│   │   ├── i18n/          # en.i18n.json, pt.i18n.json
│   │   └── images/
│   ├── routes/            # GoRouter config (app_router.dart) + AppRoutes
│   └── theme/             # AppColorsExtension, AppTextStyles, AppTheme
├── core/
│   ├── constants/         # AppConstants (XP, levels, storage keys)
│   ├── errors/            # CacheException, CacheFailure
│   └── utils/             # Extensions (Context, String, DateTime)
├── features/
│   └── <feature>/
│       ├── data/
│       │   ├── datasources/   # Local data sources (in-memory)
│       │   ├── models/        # Freezed models + toEntity()/toModel()
│       │   └── repositories/  # Repository implementations
│       ├── domain/
│       │   ├── entities/      # Plain Dart classes, no external dependencies
│       │   ├── repositories/  # Abstract contracts
│       │   └── usecases/      # One use case per class
│       └── presentation/
│           ├── cubit/         # Cubit + State (sealed classes)
│           ├── pages/         # Full-screen widgets (StatelessWidget)
│           └── widgets/       # Reusable UI components
├── gen/
│   └── i18n/              # Auto-generated by slang (do not edit)
└── main.dart              # Root BlocProvider + App bootstrap
```

### Existing Features

| Feature | Description |
|---|---|
| `landing` | Welcome screen with app introduction |
| `home` | Main hub — XP bar, streak, daily challenge, learning path with level cards |
| `quiz` | Quiz session per level — 3 question types (multipleChoice, fillInTheBlank, codeBased) |
| `progress` | Progress dashboard — level badge, XP, streak, statistics |

### Data Flow — Always respect these layers

```
UI (Page/Widget) → Cubit → UseCase → Repository (abstract) → DataSource
```

- **Never** skip layers (e.g., never call a DataSource directly from a Cubit).
- **Domain layer has zero dependencies** on data or presentation.
- Cubits depend on **UseCases**, never on Repositories directly.

## State Management — Bloc/Cubit

- Use **`Cubit<State>`** for simple state (most cases in this project).
- States must be **`sealed class`** with subclasses for each state (Initial, Loading, Loaded, Error).
- Pages use **`BlocBuilder`** or **`BlocProvider`** to consume state.
- Keep Cubits **thin**: business logic belongs in **UseCases**.
- Avoid `setState` except for trivial UI-only animations.

```dart
// Cubit
class ProgressCubit extends Cubit<ProgressState> {
  ProgressCubit({
    required GetUserProgress getUserProgress,
    required UpdateXp updateXp,
  }) : _getUserProgress = getUserProgress,
       _updateXp = updateXp,
       super(ProgressInitial());

  final GetUserProgress _getUserProgress;
  final UpdateXp _updateXp;

  Future<void> loadProgress() async {
    emit(ProgressLoading());
    try {
      final progress = await _getUserProgress();
      emit(ProgressLoaded(progress));
    } on Exception catch (e) {
      emit(ProgressError(e.toString()));
    }
  }
}

// State — sealed class
sealed class ProgressState {}
final class ProgressInitial extends ProgressState {}
final class ProgressLoading extends ProgressState {}
final class ProgressLoaded extends ProgressState {
  ProgressLoaded(this.progress);
  final UserProgress progress;
}
final class ProgressError extends ProgressState {
  ProgressError(this.message);
  final String message;
}
```

```dart
// In a page
class ProgressPage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return BlocBuilder<ProgressCubit, ProgressState>(
      builder: (context, state) {
        if (state is ProgressLoaded) return _buildContent(state.progress);
        if (state is ProgressLoading) return const CircularProgressIndicator();
        if (state is ProgressError) return Text('Error: ${state.message}');
        return const SizedBox.shrink();
      },
    );
  }
}
```

### Important pattern: `QuizState` uses manual `copyWith` with nullable functions

`QuizState` is not sealed — it uses a single class with `copyWith` accepting `Function()?` for nullable fields:

```dart
QuizState copyWith({
  int? Function()? selectedAnswer,  // () => null to clear, () => value to set
  ...
})
```

## Dependency Injection — Factory Constructors

Each Cubit has a **`factory .create()`** that wires the entire dependency tree:

```dart
factory QuizCubit.create() {
  final datasource = QuizLocalDatasource();
  final repository = QuizRepositoryImpl(datasource);
  return QuizCubit(
    getQuestionsByLevel: GetQuestionsByLevel(repository),
    submitAnswer: const SubmitAnswer(),
  );
}
```

- `BlocProvider` in the widget tree calls `.create()` and manages the lifecycle.
- `ProgressCubit` is provided in **`main.dart`** (global), while `QuizCubit` is provided per route.
- The named constructor receives dependencies as parameters (testable).

## Routing — GoRouter

- Declare all route paths as **static constants** in `AppRoutes`.
- Use **`context.go()`** for top-level navigation, **`context.push()`** to push on stack.
- Dynamic routes use path parameters (e.g., `/quiz/:level`).

```dart
abstract class AppRoutes {
  static const String landing = '/';
  static const String home = '/home';
  static const String quiz = '/quiz/:level';
  static const String progress = '/progress';
  static String quizByLevel(int level) => '/quiz/$level';
}
```

## Models & Entities

### Entity (domain/) — Plain Dart
- No annotations, no external dependencies. Immutable with `const` constructor and manual `copyWith`.

### Model (data/models/) — Freezed
- Annotated with `@freezed`. Generates `fromJson`/`toJson` via `json_serializable`.
- **Always** provide mapper extensions: `toEntity()` and `toModel()` (separate extensions in the same file).

```dart
extension QuestionModelMapper on QuestionModel {
  Question toEntity() => Question(id: id, level: level, ...);
}

extension QuestionToModel on Question {
  QuestionModel toModel() => QuestionModel(id: id, level: level, ...);
}
```

After changing a model, run:
```bash
dart run build_runner build --delete-conflicting-outputs
```

## Theming

- **Dark-only** Material 3 via `AppTheme.dark`.
- **Colors**: Access via `context.appColors` (extension method on `BuildContext` returning `AppColorsExtension`). Never use raw `Color(0x...)` in widgets.
- **Text styles**: Use `AppTextStyles.heading1`, `.bodyMedium`, `.code`, etc. Never inline `TextStyle` in widgets.
- **Palette**: primary (neon green), secondary (cyan), accent (amber), gamification (xpGold, streakFlame, levelBadge), code editor (codeBackground, codeKeyword, codeString, codeComment).

```dart
// Correct
final colors = context.appColors;
Text('Hello', style: TextStyle(color: colors.primary));

// Wrong
Text('Hello', style: TextStyle(color: Color(0xFF4ADE80)));
```

## i18n — Slang

- Source files: `lib/app/assets/i18n/en.i18n.json` and `pt.i18n.json`.
- Access translations via **`context.t`** (e.g., `context.t.quiz.checkAnswer`, `context.t.common.xp(count: 50)`).
- Dynamic parameters use `$variable` in JSON (e.g., `"$current of $total"`).
- After editing JSON files, regenerate with `dart run slang`.
- Generated files live in `lib/gen/i18n/` — **never edit manually**.

## Coding Conventions

| Rule | Detail |
|---|---|
| **Naming** | `snake_case` files, `PascalCase` classes, `camelCase` members |
| **Imports** | Always `package:flutter_base/...`, never relative |
| **Contracts** | `abstract class` for repositories and datasources |
| **`const`** | Apply `const` constructors and values wherever possible |
| **Private fields** | Prefix with `_`; expose only what's needed |
| **File length** | Keep under 200 lines; split if larger |
| **Widgets** | Extract to `widgets/` when exceeding ~50 lines or reused |
| **Private widgets** | Extract as `class _WidgetName extends StatelessWidget` in the same file when used only locally |
| **Lint** | Follow `very_good_analysis`; never add `// ignore` without justification |

## Adding a New Feature

1. Create folder structure under `lib/features/<feature>/` with `data/`, `domain/`, `presentation/`.
2. Define the **entity** in `domain/entities/` (plain Dart, immutable).
3. Create the **repository contract** in `domain/repositories/` (abstract class).
4. Implement **use cases** in `domain/usecases/` (one class per use case, receives repository in constructor).
5. Create the **model** (Freezed) in `data/models/` with `toEntity()` / `toModel()` mapper extensions.
6. Implement the **datasource** (local only) in `data/datasources/`.
7. Implement the **repository** in `data/repositories/`.
8. Create **Cubit + State** in `presentation/cubit/` with factory `.create()`.
9. Add **route** in `app_router.dart` and constant in `AppRoutes`.
10. Build **pages** (`StatelessWidget` + `BlocBuilder`) and **widgets** in `presentation/`.
11. Add i18n keys to `en.i18n.json` and `pt.i18n.json`, run `dart run slang`.
12. If a model was created/changed, run `dart run build_runner build --delete-conflicting-outputs`.
